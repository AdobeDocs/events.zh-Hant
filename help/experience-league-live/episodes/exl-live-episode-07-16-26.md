# Adobe Journey Optimizer中的AEM內容片段：新增功能及使用方式

在這場Experience League Live會議中，我們將逐步介紹Adobe Journey Optimizer (AJO)中Adobe Experience Manager (AEM)內容片段的最新增強功能，讓您的內容團隊和歷程從業人員能夠以更少的摩擦和更多的控制力，更有效率地共同作業。

[![ExL LIVE 2026年7月16日](assets/WebBannerExlLive-July16-2026.png)](https://youtube.com/live/lrdorB0LgFo){target="_blank"}

無論您是管理內容資料庫的AEM作者，還是AJO從業人員建立個人化歷程，本課程涵蓋新增功能及如何運作。

加入我們，進一步瞭解如何：

從多個AEM設定（包括作者、發佈和已驗證的發佈層級）擷取內容片段，並切換您在歷程和促銷活動中使用的來源存放庫。

從AJO中新的專用內容片段清單頁面管理片段生命週期，您可以在此識別不同步的片段，並觸發手動同步以保持最新狀態。

選取片段後，會在訊息中保留內容片段內容，讓作者可在內容區塊中重複使用片段欄位，不需重新選取。

使用地區設定和變數支援，以更刻意地使用相同內容片段的替代版本。

如果您在Managed Services，請直接在AJO中存取AEM內容片段。 在設定中新增AEM Managed Services存放庫URL一次，您就可以準備就緒了。


接下來您將清楚瞭解AEM和AJO如何在生產內容工作流程中搭配使用，並親手操作insight以使用新功能，讓片段更實用、更易於管理，且更易於大規模生產。

適合就內容進行共同作業並想要減少跨團隊切換衝突的AEM作者和AJO從業者。
