# 詢問專家 — 測量速度

學習使用 [!DNL Workfront] 報告。 2019年8月14日記錄了這次講習班。

>[!VIDEO](https://video.tv.adobe.com/v/341057/?quality=12)

## 演示文稿中使用的自定義欄位

通過複製和貼上下面的計算來節省一些時間。

**首次提交日期**

格式：日期

計算：

```
IF(ISBLANK(First Commit Date),Default Baseline.Planned Completion Date,First Commit Date)
```

**第一個持續時間**

格式：文本

計算：

```
IF(ISBLANK(First Duration),Default Baseline.Duration,First Duration)
```

**工作與承諾比率**

格式：數字

計算：

```
ROUND(DIV(Actual Duration,First Duration),1)
```

**工作與提交比率狀態**

格式：文本

計算：

```
IF({Work-to-Commit Ratio}>2,"Terrible",IF({Work-to-CommitRatio}>1.6,"Poor",IF({Work-to-Commit Ratio}>1.2,"Not Bad","Exc ellent")))
```

**調整的速度**

格式：數字

計算：

```
ROUND(DIV(Actual Duration,Duration),1)
```

**已調整的速度狀態**

格式：文本

計算：

```
IF(Adjusted Velocity>2,"Terrible",IF(Adjusted Velocity>1.6,"Poor",IF(Adjusted Velocity>1.2,"Not Bad","Excellent")))
```

## 問答

**問題**

您好，感謝您組織此網路研討會。 我有個問題，關於Workfront的菲爾德。 在系統中，我們建立了一個自定義欄位，叫做「狀態」，它是「狀態」和「條件」的組合。 這個州域包含了上千個項目中的許多雕像，對於我們的Tableau資料提取非常重要。 但是，現在我們要刪除此欄位，並使用Condion欄位，即本機欄位。 您知道如何在不丟失資料的情況下翻轉此欄位。 我現在只能考慮在不丟失資料的情況下將其手動從項目切換到項目。

**回答**

對於這種情況，您可以使用過濾和批量編輯來半自動化基於「狀態」自定義欄位填充「條件」欄位的任務。

步驟如下：

1. 確定要映射到條件值的狀態值。 例如，假設您的狀態值為「Late」和「Tery Late」，它們都映射為「In Trouble」的條件值
1. 建立顯示狀態值為「延遲」和「非常延遲」的所有項目的項目報告
1. 運行報告。 確保顯示所有項目（請參閱報告右下角的選項）
1. 按一下帶列標題的欄中報告左上角的複選框。 這將選擇報表中的所有項目
1. 按一下報告清單上方的「編輯」按鈕
1. 將條件類型設定為「手動」
1. 將「條件」欄位設定為「出現故障」
1. 按一下「保存更改」


**問題**

如何定義「優秀」、「不壞」等？

**回答**

這只是一個例子，但我是這麼設計的。 首先計算了兩個指標：

調整的速度

此公式為「實際工期/計畫工期」（儲存在項目的「工期」欄位中）。 由於項目的計畫工期可以在每次重新計畫項目時進行更改，因此計畫工期表示最終重新計畫。

工作與承諾比率

此公式與「已調整的速度」類似，但與其使用最終重新計畫中的「計畫工期」值，不如使用最初向客戶承諾的「計畫工期」。 我們假定原始基線包含此資訊（我們計畫從現在開始要求項目經理以這種方式規劃他們的項目，以便我們能夠捕獲準確的資料）。 我們從原始基線中捕獲了此持續時間值，並將其稱為「第一持續時間」。

通過將實際持續時間除以計畫持續時間或第一持續時間，我們最終會得到一個數字，它可以告訴我們我們離目標有多近。 如果「計畫工期」或「第一工期」等於「實際工期」，則索引將等於1。 如果「實際持續時間」大於1，則答案將大於1。 我們約會的次數越多越糟糕。

因此，考慮到我決定為「已調整速度」和「工作 — 承諾比」指定以下狀態：

* 1.1或更低，我叫Excell。
* 1.2到1.5，我稱之為不錯。
* 1.6到1.9，我叫窮。
* 2歲或2歲以上的人，我稱為「糟糕」。

我們沒有追蹤在項目上實際花費的時間我們只是追蹤和比較工期 但是，如果您正在跟蹤小時數，並且希望使用計畫小時數之間的實際小時數來計算速度，則可以通過將計畫小時數與實際小時數進行比較來執行此類報告。 您也希望從原始基線捕獲計畫工時。

**問題**

**回答**

我為Velocity報告使用了兩個過濾規則：

我還添加了其他過濾功能，使我的報告保持在足夠小的規模，可以用於網路研討會，但在您的生產環境中，您可能希望在特定時間段內查看所有使用WPI自定義表單的項目。 您可能希望篩選該項目的實際完成日期。

否，基線不包括在複製的項目中

**問題**

對於任務審批流程，是否可以向我演示如何建立報告，該報告按項目中的任務提供審核跟蹤，並為每個審批人設定時間/日期戳？

接下來轉到「篩選器」頁籤，並為以下項添加篩選器規則：

任務>>是批准>>等於>>真。 這將僅顯示附加了批准的任務。

根據需要添加其他篩選器。

**問題**

我想製作一份證據報告。 一個項目清單，其中顯示它們有多少個證明以及該證明有多少個版本。

**回答**

建立文檔報告。

預設視圖將顯示版本號。 您將希望保留該列，但可以更改任何其他列。

按項目名稱對報表進行分組。

按當前版本篩選報告：校樣ID不為空。

這會給你一個項目中所有證據的清單。 它將為每個校樣設定一行，並顯示版本號（與版本總數相同）。

**問題**

您能在任務級別使用速度嗎？ 而不是項目級別？

**回答**

是的，但您需要複製項目自定義表單並從中建立任務自定義表單。 然後，您需要在「第一次提交日期」欄位中編輯計算，並將對「預設基線」的引用更改為「預設基線任務」。 對第一次持續時間執行相同操作。 之後，您可以將任務自定義表單附加到要測量的任何任務。 您需要為這些任務建立任務報告，而不是項目報告。 但是，您仍需要確保將「原始」項目基線設定為預設基線。 所有任務資料都與項目資料保留在同一基線中。

**問題**

是否必須由項目所有者手動設定第一個提交日期？ 還是從現有領域拉出來？

**回答**

從預設基線捕獲「第一次提交日期」。 只要預設基線是原始基線，它將顯示項目首次設定為「當前」狀態時的計畫完成日期。

**問題**

是否仍需要定期刷新自定義表單中的計算欄位？ 還是會在一夜之間（或在其他觸發器上）自動發生？

**回答**

重新計算計算欄位：

* 用戶編輯對象時
* 使用已激活的重新計算自定義表達式進行批量編輯時
* 使用選定的「更新以前的計算」選項對表單的修改

**問題**

如果速度考慮工期，「項目首選項中完成百分比」是否應基於工期？

**回答**

否，「項目首選項」選項僅指「完成百分比」的計算方式。 持續時間值本身不受此設定的影響。

**問題**

第一個計畫持續時間與計畫持續時間之間有何區別？

**回答**

First Duration是您最初向客戶承諾的項目將花費的天數。 我們從項目從「計畫」更改為「當前」時記錄的原始基線中獲取此數字。

Planed Duration是從項目開始到計畫完成日期的天數。 最初，這兩個工期是相同的，但如果項目曾經重新計畫過並且計畫完成日期已更改，則計畫工期也會更改。

Velocity報告的值來自我們能夠查看計畫持續時間與第一持續時間之間的變化。 我們可以從最早開始記錄基線時就看到這一點。

**問題**

您是否能讓工作人員按顏色設定，以便在所有報告中保持相同？

**回答**

如果在任務報告中按「分配給」>>「名稱」分組，則可以在「圖表」頁籤中為特定工作人員分配顏色。 只需選擇「圖表」頁籤中「分配給」>>「名稱」框旁邊的「自定義顏色」選項，然後為每個工作人員添加顏色。

**問題**

我試圖確定是否可以建立一個儀表板，其中有一個區域查看任務層自定義表單，以查看它是否存在，如果某些欄位不為空則是輔助欄位。 這可能嗎？

**回答**

看看我是否理解你的問題。 假設我有一個名為Tammy Form的任務自定義表單，其中包含一個名為Tammy Field的欄位。

您想要一個任務報告，該報告將顯示附加了Tammy表單且其中Tammy欄位具有某些值的所有任務。

是的，你可以。 您只需要在任務報告中使用兩個篩選器規則即可：

類別>> ID等於表格

任務>> Tammy欄位不為空

**問題**

是否可以建立報告以在文檔庫中查找特定命名文檔？ 要測量某些文檔是否存在的儀表板的一部分。

**回答**

可以。您需要建立文檔報告。 似乎您每次運行報告時都希望提供特定的文檔名稱。 如果是這種情況，我建議轉到「報告選項」並選擇「報告提示」。 添加「文檔」>>「名稱」提示。

**問題**

是否可以選擇圖表頁籤中未列出的顏色/十六進位值，但該顏色是新的十六進位值，例如，我的品牌顏色中的新顏色允許我自定義報告？

**回答**

是，您可以輸入任何可能已找到的RGB代碼。 這是一個標準代碼，它指示顏色中包含的紅色、綠色和藍色的數量。 Workfront將接受從000000到FFFFFF的任何十六進位值，因此，如果您知道您的品牌顏色代碼，則可以使用它。

是否可以在儀表板中重新聲明報告的定義（提供每個報告度量的說明）?

**問題**

如何將新資料添加到x軸？

**回答**

按報表中的任何內容分組時，將允許您建立圖表。 然後，可以在「圖表」頁籤中設定X或Y軸。

**問題**

您能否查看第一個持續時間與實際持續時間之間的差異？

**回答**

First Duration是您最初向客戶承諾的項目將花費的天數。 我們從項目從「計畫」更改為「當前」時記錄的原始基線中獲取此數字。

Actual Duration是從項目開始到實際完成日期的天數。

**問題**

項目基線系數如何納入此報告？

**回答**

項目的原始基線包含項目首次更改為「當前」狀態時存在的計畫完成日期和計畫持續時間。 如果您的流程是在將項目設定為「當前」之前計畫項目，則這將表示您承諾完成項目的日期。

**問題**

是否有辦法將新表單成批應用於舊項目？

**回答**

是，您可以從清單中選擇多個項目。 執行此操作時，清單左上角將顯示「編輯」選項。 選擇多個對象時按一下「編輯」(Edit)，您將進入我們稱為「批量編輯」的內容。 可以以這種方式將自定義表單添加到多個項目中。

向大量項目添加自定義表單的快捷方式是建立一個報告，您可以篩選該報告，以僅包括所需的項目。 然後，您不必單獨選擇項目，只需按一下清單中第一個複選框上方的複選框，即可選擇整個清單。

**問題**

是否可以從分配報表的分組中刪除重複條目，但不能跨分組刪除重複條目？

**回答**

在清單報告中考慮分組的最佳方式是：

首先，使用「篩選器」(Filter)頁籤控制清單中顯示的項目。 將不存在重複的條目。 該篩選器將應用於每個對象。 如果它通過過濾器，它將在清單中顯示一次，如果它不通過，它將根本不顯示。

下一個分組將應用到篩選的清單。 分組標識清單中對象的一項內容，如其所在組合的名稱（您不能在事物清單上分組，只能在一項內容上分組）。 然後，具有相同值的所有對象都將出現在該分組中，就像同一組合中的所有項目一樣。 未選擇項目組合的任何項目都將出現在名為「無值」的分組中。

因此，任何對象都不可能出現在多個分組中。 而且，對象是否出現在清單中完全由篩選器控制（以及運行報表的人員是否有權查看它）。

**問題**

是否建議使用其他報告來跟蹤Velocity? 只是高層推薦是件好事，因為我知道沒有足夠的時間去詳細地討論。

**回答**

和任何報告一樣，你需要做的第一件事就是決定你想知道什麼。 下一步是訪問這些資訊，這在某些情況下意味著你需要開始跟蹤它。

我決定將實際持續時間與兩種計畫持續時間進行比較的一個原因是我認為它提供了關於速度的有趣見解。 資料也已經可用了，所以我不必開始跟蹤它。 通過幾次計算，我可以以我可以報告的形式提取資料。

但您最好還是決定跟蹤其他有關要報告的任務或項目的資訊。

Workfront沒有任何內置的速度報告，因此我建議您和您的團隊集思廣益，瞭解您想要知道什麼來確定速度，然後查看您需要跟蹤什麼。

**問題**

您能在COLUMN級別計算任何內容嗎？ 而不是從自定義表單調用計算的FIELD?

**回答**

可以在文本模式下使用值表達式來執行這些計算。 我們無法完成「第一持續時間」或「第一提交日期」，但我們需要在他們不會更改的位置捕獲這些資料。

對於「工作與提交比率狀態」和「已調整速度狀態」，這些欄位需要是自定義欄位，以便我們可以在「圖表」頁籤中使用它們。 「圖表」頁籤不識別文本模式分組，它們需要是自定義欄位。 由於我們需要「工作 — 提交比」(Work-to-Commit Ratio)和「調整速度」(Adjusted Velocity)來計算這些狀態，因此我們也需要它們作為自定義欄位。 因此，在這種情況下，它們都需要是定制欄位，但總是很好的，可以同時考慮兩種方式，並選擇哪種方式最好。 謝謝問題。

**問題**

我們的項目經常因為客戶的延誤或整個過程的變化而改變。 我們的報告可能都顯示「糟糕」。 什麼是用於跟蹤更改原因的建議？ 我們考慮在文檔級別添加一個自定義表單，該表單報告更改原因（內部或客戶端更改），但想知道什麼是最佳做法。

**回答**

最佳做法是使用下拉菜單來跟蹤此操作。 請在此處添加盡可能多的「原因」，然後添加一個「其它」選項以捕獲不在清單中的原因。 如果新原因看起來或變得常見，請將其添加到下拉清單中。 您可以輕鬆地在下拉清單中報告內容，也可以在此欄位上分組（不能在複選框或多選下拉清單上分組）。

再談一下。 您可能不想在Velocity報告中包括所有項目。 如果你在修補漏洞或者「去以前沒人去過的地方」，你大概不會像建造一棟你以前建過很多次的房子一樣，對完工日期做出同樣的承諾。

因此，請確保將速度報告重點放在可以幫助您實現目標的位置。

**問題**

如果我將項目的預設基線設定為「原始」，然後將項目從當前項目中取出並放回當前項目，它是否會更改我的預設基線？

**回答**

可以。每次將狀態更改為「當前」時，您都會得到新的基線，並且它將是新的預設值。 但所有以前的基線仍然存在，您可以手動將原始基線再次設定為預設基線。

**問題**

是否有方法在報告中設定可編輯的欄位？ 是否可以為某些欄位設定限制？

**回答**

您可以限制自定義窗體中欄位的查看和編輯權限。 您需要將欄位包括在分區中，而在分區設定中，您可以選擇用戶查看或編輯分區中的欄位所需的權限。

**問題**

是否可以建立在文檔庫中查找特定命名文檔的報告？

**回答**

可以。您需要建立文檔報告。 似乎您每次運行報告時都希望提供特定的文檔名稱。 如果是這種情況，我建議轉到「報告選項」並選擇「報告提示」。 添加「文檔」>>「名稱」提示。

**問題**

在報告中，為什麼值可用作列，但不可用於選擇或分組。 例如：問題源。

**回答**

列可能可查看但無法進行分組的主要原因是它可能包含清單，如複選框自定義域或任務分配。 不允許對清單進行分組。

**問題**

如何在發生小時輸入和實際執行小時時在報告（按哪些欄位）中分隔？

**回答**

「工時」對象的「錄入日期」欄位引用了工時的工作日期。 這使「輸入日期」(Entry Date)與其他對象不同，即表示對象的建立日期。 即使沒有小時的建立日期，也存在「上次更新日期」，該日期最初是建立日期，然後是小時之後編輯的任何日期。

**問題**

從報告角度看，我們如何訪問基線資料？ 我有一個項目，有多個基線。 我想瞭解在每個基線中如何計畫各個任務。 是否有一種方法來編寫報告，以顯示每個基線的項目計畫？

**回答**

報告將顯示當前設定為預設值的基線要查看的任何欄位，以便您可以更改基線並刷新報告以查看具有新基線的欄位。

但是，如果您想以圖形方式查看有關任務的資訊，可以使用甘特圖功能來完成。 在任務清單中開啟甘特圖（使用「自動保存」旁右上角的甘特表徵圖），然後轉到正下方和右側的「設定」表徵圖，然後按一下它。 選中「基線」(Baseline)框，它將顯示所有基線。 您可以一次選擇這些任務，並在甘特圖視圖中查看任務中的更改。

**問題**

如何建立報表，以查找已定義期間（例如上個月）的狀態更改。

**回答**

Workfront的「分析」功能為您提供了查看歷史資料（包括狀態更改）的簡便方法。

但您也可以使用「備注」報告獲取狀態更改資訊。 如果要跟蹤「項目狀態」欄位，則可以進行篩選以查看項目的狀態更改。

首先，轉到「設定」>「介面」>「更新源」，並確保「項目狀態」是正在跟蹤的內置欄位之一。 如果不是，您需要添加它。

現在建立「注釋」報告並執行以下操作：

在列（視圖）頁籤中：

* 替換「審計文本」的「注釋文本」列。 這將顯示狀態從和更改到的資訊
* 離開「項目：名稱」和「輸入日期」列
* 按一下「Entry Date」列，然後在「Column Settings」（列設定）面板中選中「Sort by this column」（按此列排序）。 如果希望查看最近的狀態更改，請按降序排序。

在「分組」頁籤中：

* 按項目分組：名稱

在「篩選器」頁籤中，建立以下篩選器規則：

* 注意>>審計類型>>等於>>狀態更改
* 添加任何附加規則以按附註錄入日期進行篩選。 您可能希望將此內容排除在篩選器之外，而改用報告提示
* 根據需要篩選項目、項目組合或其他資料。

**問題**

作為計畫員，您能否為其他用戶提取報表？

**回答**

計畫員可以建立報表，並與任何用戶共用，即使不是用戶。 查看報告時，請轉至「報告操作」>「共用」，然後按一下「報告訪問」框右上角的齒輪表徵圖。 選擇「將此公開給外部用戶」。 的雙曲餘切值。 您可以複製提供的連結並將其發送給任何人。 他們將在瀏覽器中即時看到報告。
