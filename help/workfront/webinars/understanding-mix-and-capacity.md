# 咨詢專家 — 瞭解組合和容量

學習衡量企業內的混合和容量。 本網路研討會於2019年10月2日錄制。

>[!VIDEO](https://video.tv.adobe.com/v/341119/?quality=12)

## 問答

**問題**

如何將%s放在柱形圖上？

**回答**

「混合報表」中列出的%值存在，因為在圖表頁籤中，我們選擇了「組列」和「堆積到100%」。 如果我們選擇「堆積」，則顯示計畫小時總數，而不是百分比。

**問題**

如果您的部門/組織工作量是項目/任務和問題/請求的混合體，您建議如何(在Workfront報告中)對WPI進行高級別審查。

**回答**

每個項目、任務和問題都需要有各自的報告和自己的自定義表單。 但是，它們都可以使用相同的「工作類型」欄位。 您可能希望在一個儀表板中顯示這三個報告。

**問題**

我們是否必須批量編輯任務，以使其能夠運行或具有戰略性？

**回答**

我們建議的方法是建立一個報告，以便您獲得可操作的任務清單。 一旦您能夠同時選擇所有任務，請批量編輯這些任務，然後將自定義表單與「工作類型」相連，並將所有任務的工作類型一次設定為「可操作」。 您將遵循同一採購以收集戰略任務清單、批量編輯這些任務並添加自定義表單等。

網路研討會中提到一些可幫助您獲取清單的自定義提示，如檢查任務名稱、項目所有者、項目組合或分配的用戶中的某些詞。 這些只是想法。 你應該設計一份最適合你情況的報告。

**問題**

如果我的組合中有4個類別，我是否可以為每個類別建立一個目標，然後報告預測與計畫之間的差值與實際值之間的差值？ （4類市場活動、業務單元、Web和產品）。 我們在項目級別以自定義窗體/欄位的形式提供類別。 目標是建立季度預測（預算/預測），然後跟蹤計畫小時數，最終實現實際。

**回答**

如果您在一個自定義欄位中具有所有類別（現在讓我們將其稱為工作類型），則只需先在「計畫小時數」和「工作類型」上建立項目報告分組。 篩選項目報表以在所需日期範圍內顯示Planning中的項目。 如果要查看百分比，請使用將列或條形堆疊成100%的圖表。 這可能是您的Forecast報告。

您可以複製報告並對其進行編輯，以基於當前項目建立報告，同時仍顯示基於計畫小時數的混合。

您可以再次複製報表，將其更改為按實際小時數而不是計畫小時數分組，並只顯示在所需日期範圍內的已完成項目。 這將顯示基於實際工時的百分比組合。

**問題**

如果您在項目或任務上有多個類別ID，此功能是否可行？

**回答**

是的，如果您有多個ID，則需要用頁籤分隔這些ID，如下所示：

```
EXISTS:1:$$EXISTSMOD=NOTEXISTS
EXISTS:1:$$OBJCODE=OBJCAT
EXISTS:1:categoryID=5d76d82600e7926bb51eeb1e0886810e 5d54288d01034619f2eb2c74f6472f18
EXISTS:1:objID=FIELD:ID
```

用制表符分隔它們的最佳方法是首先在生成器中建立類別清單。 放入多個自定義表單名稱，當您切換到文本模式時，將視它們為由制表符分隔的ID。

多個ID被視為OR，因此如果其中任何一個ID附加到任務，則它不會顯示在報告中。

**問題**

報告是提示「ANDed」還是「ORed」？

**回答**

單個自定義提示為「ANDed」。 因此，如果將Pam指定為項目所有者，將Bill指定為分配給任務，則您將只看到Pam為項目所有者的項目中分配給Bill的任務。

**問題**

為什麼只能按特定列排序？ 即不能按任務排序。

**回答**

「工作總攬」是一個清單，您不能對項目清單進行排序或分組。 您只能對單個項目進行排序或分組。

為了說明這一點，請想像一個任務上的任務清單：

```
Jane
Bill
Dan
```

此外還列出了另一個任務上的分配清單：

```
Bill
Jane
Helen
```

哪項任務應先出現在排序中？ 你可以說「按清單中的名字排序」，但這並非必要有用，因為你無法控制順序。 Workfront完全不允許你按清單排序，從而避免了問題。

按清單分組呢？ 如果我們可以按名稱清單進行分組，您會發現分配給Jane、Bill、Dan的所有任務以及分配給Jane、Dan、Bill的所有任務（同一清單，但按不同順序）在不同的分組中。 同樣，Workfront不允許按清單分組，從而避免了這一問題。

**問題**

計畫小時數是否用於戰略任務和實際操作小時數？

**回答**

不可以。 在我們的示例中，我們使用「計畫小時數」來顯示為戰略任務和操作任務計畫的工作量水準。 這使我們能夠很容易地比較它們，無論是過去、現在還是將來。 您還可以使用實際小時數來比較戰略任務和操作任務，但僅用於過去的任務，因為實際小時數是人們報告的小時數，作為他們實際處理任務所花的時間。

**問題**

在資源計畫器中，如何對過去計畫但未完成的任務進行計算？ 計畫的工時似乎沒有向前滾動，因此，未來幾週內不會顯示為需要資源的任務/工時。

**回答**

沒有「自動」向前滾動未完成的工作。 發生此情況時，您需要重新計畫項目。 也許您最初分配給某個任務的資源在新的時間框架內不可用，因此項目經理必須查看此資訊並決定重新規劃的最佳方式。 這可能意味著讓利益相關方參與並獲得對計畫更改的批准。

**問題**

您是否建議調整其FTE，而不是每天輸入2小時來檢查電子郵件、中斷？

**回答**

是，如果將FTE設定為。75，則資源規劃器中將用戶顯示為每天6小時可用。 這將是他們每天的可用性。 如果要根據日期將它們顯示為不同期間不可用，例如每個季度的最後一天不可用，則製造間接費用項目就是執行此操作的方法。

有些人更喜歡間接費用項目，因為他們可以為自己構建項目並根據需要進行更改，而他們可能無權編輯自己的FTE。

**問題**

團隊容量控制板的資料是否可供與您共用報告的任何人使用，而不管他們在工作中擁有什麼權限？

**回答**

如果用戶沒有查看對象的權限，則在報表/儀表板中將看不到該對象。 但是，如果希望每個人都看到相同的結果，可以進入「報告操作」>「編輯」>「報告設定」，然後在您的名稱中輸入「運行此報告，並具有的訪問權限」。 這將允許在此報告上共用的用戶查看您看到的確切結果。 它不會授予他們對結果本身的額外訪問權限，因此某些結果可能可以按一下，也可能不可按一下。

**問題**

我們如何建立一個報告來顯示分配給整個項目的所有員工（不是在任務級別）?

**回答**

您可以在「項目報告」中建立一列，該列顯示作為「人員配備」頁籤（「項目團隊」）一部分列出的所有用戶。 您要使用以下文本模式：

```
displayname=Project Team
listdelimiter=<p>
listmethod=nested(projectUsers).lists
textmode=true
type=iterate
valuefield=user:name
valueformat=HTML
```

**問題**

我希望有一個報告/儀表板，其中包含我的團隊的工作方式。 特別是以下情形： — 我擁有的項目/為我建立的項目/我分配給其他人的任務/分配給我的任務

**回答**

我擁有的項目

有一個名為「當前項目」的內置報表，它將顯示所有當前項目。 您可以編輯此項目並添加篩選器規則：項目>所有者ID >等於> $$USER.IDT保存並將報表更名為「項目我擁有」。

為我建立的項目

有一個名為「我的項目」的內置報表，它將顯示您在項目團隊中的所有當前項目（即您是所有者、發起人或分配給任務）。 不確定這是否是您要求的，但除了讓您成為項目所有者、發起人或將您分配給任務外，沒有其他方法可以瞭解是否有人為您建立了項目。

我分配給其他人的任務

使用您想要的任何篩選器建立任務報告，然後轉到「篩選器」頁籤，然後按一下「切換到文本模式」。 將此代碼添加到已存在的任何代碼：

```
EXISTS:1:$$OBJCODE=ASSGN
EXISTS:1:taskID=FIELD:ID
EXISTS:1:assignedByID=$$USER.ID
```

這將顯示登錄用戶分配了至少一個當前受分配人的所有任務。 如果受分配人由多人分配，則僅在任務登錄頁上，分配某人的第一個人的姓名將顯示為「請求者」。 如果您想查看所有已分配的人員以及分配了每個人員的人員，您可以向視圖中添加一列，切換到文本模式，然後用以下內容替換任何人員：

```
displayname=All Assignees and Requesters
listdelimiter=<p>
listmethod=nested(assignments).lists
textmode=true
type=iterate
valueexpression=CONCAT("Assigned To: ",{assignedTo}.{name},"; Requested By: ",{assignedBy}.{name})
valueformat=HTML
```

分配給我的任務

有一個名為「我的任務」的內置報告，它將顯示您作為任務所有者的當前項目上所有未完成的任務。 我建議您更改篩選器，以顯示您所分配的所有任務，其中您是可能分配的眾多用戶之一，而不只是任務所有者。 通過刪除篩選器規則來執行此操作

```
Task > Assigned To ID > Equal > $$USER.ID 
```

並替換成

```
Assignment Users > ID > Equal > $$USER.ID
```

**問題**

是否有方法自定義圖表上的標籤？ 我發現，當我建立一個圖表以反映項目狀態時 — 最終，標籤中包含了家庭組的名稱。

**回答**

圖表上的標籤使用您要分組的欄位名稱。 因此，您更改標籤的唯一方法是使用計算的自定義欄位，該欄位包含您想要的任何名稱。 在欄位的計算部分中，輸入要分組依據的本機欄位的欄位名。

**問題**

請如何對查克的團隊分配報表欄進行顏色編碼？ 暗指琥珀色，暗指紅色，準時指綠色？ 您能否將順序更改為更符合邏輯，即紅色/琥珀色/綠色還是相反？

**回答**

要更改「進度狀態」選項在報告中使用的顏色，請編輯報告，然後按一下「圖表」頁籤。 查找「自定義顏色>」下拉框。 它將顯示在「左(Y)軸」框或「資料分組依據」框的旁邊，具體取決於您是否選擇分組列或條形。 在該下拉框中，您可以選擇顏色。 如果按一下顏色選項右下角的數字，則可以從較大的調色板中選擇顏色。

很遺憾，您不能更改這些順序。

**問題**

是否可以建立一個圖表，其中它指向為工作人員分配任務的項目計數？

**回答**

是的，這是方法：

* 建立項目報告
* 如果所詢問的用戶是登錄用戶，則篩選器應包括以下行：

```
   Project Users > ID > Equal >$$USER.ID 
```

* 否則，將用戶名替換 [!DNL $$USER.ID]。 這將顯示為此人分配了任務或是所有者或發起人的所有項目。 如果您只想查看分配任務的項目，則應添加以下兩個附加篩選器規則：

```
   Project > Owner ID > Not Equal > $$USERID
   Project > Sponsor ID > Not Equal > $$USERID
```

* 至少建立一個分組，以便您可以建立圖表。 組成任何組，比如公司。 然後按一下「圖表」頁籤並選擇圖表。 「記錄計數」將是一個軸的預設值。 這將是用戶在中分配的項目數。

當用戶在項目（分配給任務或項目所有者或項目贊助商）上獲得分配時，該人員將添加到項目團隊，並可在「人員」子標籤下的「人員配備」標籤中看到。 如果用戶被從項目所有者、贊助商或任何任務分配中刪除，則其名稱仍在項目團隊中。 如果要刪除它，必須手動將其刪除。 請記住，這可能會影響報告結果的準確性。 要從項目團隊中刪除某人，請轉至「人員配備」>「人員」，選擇該人員或人員，然後按一下清單上方顯示的「刪除」按鈕。

**問題**

如何在文本模式（在分組中）中更改列的降序順序？

**回答**

生成報表時，可以選擇在「列（視圖）」頁籤中對大多數列進行排序。 如果您沒有分組，將對整個清單報告進行排序。 如果您有分組，它將根據每個分組中的該選項進行排序。

**問題**

如何建立列，以標識分配給審批階段的團隊成員？

**回答**

如果您運行的是「任務」或「發出/請求」報告，則Report Builder中有一列稱為「批准者和狀態」，它將納入此資訊。
